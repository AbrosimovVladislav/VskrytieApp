# Шаг 1 — Поиск матча

**Источник**: Perplexity API (3 параллельных запроса + верификация)
**Промпт**: [`prompts/find-match.md`](../prompts/find-match.md)
**Эндпоинт**: `POST /api/find-match` (отдельно от pipeline)

## Вход

```ts
{
  query: string;   // запрос пользователя ("ЦСКА", "Спартак Динамо")
  sport: string;   // из UI
  league: string;  // из UI (id лиги)
}
```

## Логика

### Раунд 1: Параллельный поиск

Запускаются **3 независимых агента** в Perplexity с разными формулировками:
1. Прямой поиск по расписанию
2. Поиск по календарю/расписанию лиги
3. Поиск по новостям и анонсам

Каждый промпт содержит **сегодняшнюю дату и текущее время МСК**.

### Фильтрация по времени

Все кандидаты проходят фильтр `isMatchInFuture`:
- **Дата > сегодня** → подходит
- **Дата < сегодня** → отбрасывается
- **Дата = сегодня** → сравниваем время начала матча с текущим временем МСК:
  - Матч ещё не начался → подходит
  - Матч уже начался или идёт → отбрасывается
  - Время неизвестно → отбрасывается (на всякий случай)

### Верификация AI

Если после фильтрации осталось несколько кандидатов — дополнительный запрос в Perplexity выбирает наиболее подходящий.

### Раунд 2: Повторный поиск

Если раунд 1 не дал результатов — запускается retry с усиленным промптом. Результат проходит ту же фильтрацию по времени.

### Fallback

Если ничего не найдено — собираем `suggestions` из всех ответов и возвращаем `MatchClarification`.

## Выход

```ts
interface MatchInfo {
  team1: string;
  team2: string;
  date: string;      // "2026-03-25"
  time: string;      // "19:30 МСК"
  venue: string;     // "Арена ЦСКА"
  league: string;    // "КХЛ"
}
```

## Если не удалось определить

Возвращаем пользователю уточняющий вопрос с вариантами (если есть).

```ts
interface MatchClarification {
  message: string;
  suggestions?: string[];
}
```
