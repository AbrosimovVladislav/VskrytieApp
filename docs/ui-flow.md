## Screen 1 — Home / Search

**Цель**: быстро принять запрос, создать ощущение "аналитика на связи".

```
┌────────────────────────────────┐
│  ВСКРЫТИЕ            [⚡ 5]    │  ← заголовок + остаток отчётов
├────────────────────────────────┤
│                                │
│   Введите команду или матч     │  ← subtitle
│                                │
│  ┌──────────────────────────┐  │
│  │ 🔍  ЦСКА — Спартак       │  │  ← поле ввода (большое, touch-friendly)
│  └──────────────────────────┘  │
│                                │
│  ┌──────────────────────────┐  │
│  │     АНАЛИЗИРОВАТЬ        │  │  ← CTA кнопка, полная ширина, accent-цвет
│  └──────────────────────────┘  │
│                                │
│  Примеры запросов:             │  ← подсказки, исчезают после первого ввода
│  · Реал Мадрид завтра          │
│  · Манчестер Сити — Арсенал    │
│  · ЦСКА хоккей                 │
│                                │
│  ── Недавние ──────────────    │  ← 3 последних запроса (если есть история)
│  ЦСКА — Локомотив    вчера     │
│  Барселона — ПСЖ    3 дня     │
│                                │
└────────────────────────────────┘
│  🏠 Главная │ 📋 История │ ⭐ Баланс │ ⚙️ Настройки  │  ← bottom nav
```

**Детали:**
- Поле ввода: `min-h-[52px]`, радиус `rounded-2xl`, фон `bg-bg-card`
- CTA кнопка: `bg-accent`, `text-bg`, `font-semibold`, `rounded-2xl`, `h-14`
- `⚡ 5` — иконка молнии + число оставшихся отчётов, тап → переход на Balance
- Подсказки-чипы с запросами — горизонтальный скролл
- Недавние запросы: тап → автозаполняет поле

## Screen 2 — Analyzing (Загрузка / Стриминг)

**Цель**: показать, что работа идёт, снять тревогу ожидания 15–20 сек.

```
┌────────────────────────────────┐
│  ←  ЦСКА — Спартак             │  ← back button (Telegram) + запрос
├────────────────────────────────┤
│                                │
│         ⚡ ВСКРЫТИЕ            │  ← лого / иконка по центру
│                                │
│  ████████████░░░░░░░░  62%     │  ← прогресс-бар
│                                │
│  ✓  Определяем матч            │  ← шаг 1: выполнен (зелёный чек)
│  ✓  Собираем статистику        │  ← шаг 2: выполнен
│  ●  Анализируем данные...      │  ← шаг 3: текущий (пульсирующая точка)
│  ○  Формируем отчёт            │  ← шаг 4: ожидает (серая точка)
│                                │
│  ┌──────────────────────────┐  │
│  │  ░░░░░░░░░░░░░░░░░░░░░░  │  │  ← skeleton секции начинают появляться
│  │  ░░░░░░░░░░░░░░░░░░░░░░  │  │     по мере стриминга
│  │  ░░░░░░░░░░░░░░          │  │
│  └──────────────────────────┘  │
│                                │
└────────────────────────────────┘
```

**Стриминг-поведение (прогрессивное раскрытие):**

| Этап | Событие SSE | Что показываем |
|---|---|---|
| 1 | `step:identify` | "Определяем матч" ✓ |
| 2 | `step:stats` | "Собираем статистику" ✓, skeleton Header |
| 3 | `step:analyze` | "Анализируем данные" ●, skeleton Form |
| 4 | `section:header` | Header секция появляется (fade-in) |
| 5 | `section:form` | Form секция появляется |
| 6 | `section:stats` | Stats секция появляется |
| 7 | `section:injuries` | Injuries секция появляется |
| 8 | `section:h2h` | H2H секция появляется |
| 9 | `section:odds` | Odds секция появляется |
| 10 | `section:recommendation` | AI-рекомендация + конфетти-haptic |

**Детали:**
- Прогресс-бар: `bg-accent`, анимированный fill
- Шаги: ✓ зелёный, ● `animate-pulse` accent, ○ muted
- Skeleton карточки: `bg-bg-card animate-pulse rounded-xl`
- Если ошибка → экран Error с кнопкой "Попробовать снова"

### 3.1 — Форма команд

```
┌────────────────────────────────┐
│  📊 ФОРМА (последние 5)        │  ← заголовок секции
├────────────────────────────────┤
│                                │
│  ЦСКА     ● ● ● ○ ●   4П 1П   │  ← зел/красн/серые dots + счётчик

│  Спартак в гостях: ○ ● ○ ● ●  │
│                                │

```

**Dots легенда:**
- `●` зелёный `#00C853` — победа
- `●` серый `#4a4a5a` — ничья
- `●` красный/оранжевый `#FF6D00` — поражение
- Размер dot: 12×12px, gap 4px
- Тап по dot → tooltip с результатом (`ЦСКА 2:0 Зенит, 15 мар`)

## Микровзаимодействия и Haptics

| Момент | Haptic | Описание |
|---|---|---|
| Нажатие "Анализировать" | `impactOccurred('light')` | Мягкий отклик |
| Секция появляется в стриминге | `impactOccurred('light')` | Каждая секция |
| Отчёт полностью готов | `impactOccurred('medium')` | Финальный аккорд |
| Ошибка | `notificationOccurred('error')` | Вибро-ошибка |
| Успешная оплата Stars | `notificationOccurred('success')` | Подтверждение |
| Тап по dot формы (tooltip) | `selectionChanged()` | Лёгкий селект |

## Анимации

| Элемент | Анимация |
|---|---|
| Секция появляется | `fade-in + slide-up`, 200ms ease-out |
| Skeleton → контент | `fade-in`, 150ms |
| Typing в рекомендации | `cursor blink`, 500ms interval |
| Прогресс-бар | `width transition`, 300ms linear |
| Карточка отчёта (history) | `scale 0.98 → 1.0` on tap |
