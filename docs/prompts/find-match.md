# Промпт: Поиск матча (шаг 1)

**API**: Perplexity (3 параллельных запроса)

## Стратегия

Запускаются 3 агента с разными формулировками для повышения надёжности. Каждый промпт содержит текущую дату и время МСК.

## Шаблон (общая структура)

```
Сегодня {today}, сейчас {nowMsk} МСК.
Матч подходит ТОЛЬКО если он ещё НЕ начался.
[Вариация запроса для каждого агента]
Найди ближайший предстоящий матч "{query}" в {league.name} сезона {league.season}.
```

### Агент 1 — Прямой поиск
Ищет матч напрямую по расписанию.

### Агент 2 — Календарь лиги
Фокус на расписании/календаре конкретной лиги.

### Агент 3 — Новости и анонсы
Ищет в новостях, анонсах, превью матчей.

## Переменные

| Переменная | Источник |
|---|---|
| `{today}` | `new Date().toISOString().split("T")[0]` |
| `{nowMsk}` | Текущее время в Europe/Moscow, "HH:MM" |
| `{query}` | Запрос пользователя |
| `{league.name}` | `LeagueConfig.name` |
| `{league.season}` | `LeagueConfig.season` |

## Формат ответа (JSON)

```json
{
  "found": true,
  "team1": "команда хозяев",
  "team2": "команда гостей",
  "date": "YYYY-MM-DD",
  "time": "HH:MM МСК",
  "venue": "арена",
  "message": "если не найден — почему",
  "suggestions": ["вариант 1", "вариант 2"]
}
```

## Верификация

Если несколько агентов нашли разные матчи — дополнительный запрос в Perplexity выбирает лучший вариант по критериям: ближайший по дате, соответствует запросу, ещё не начался.

## Серверная фильтрация

Независимо от ответов Perplexity, сервер проверяет `isMatchInFuture`:
- Дата в будущем → ок
- Дата сегодня, время начала > текущее время МСК → ок
- Иначе → отбрасывается
