# Backend — Оркестратор

Оркестратор управляет последовательностью шагов pipeline. Не знает что внутри каждого шага — передаёт данные между ними.

## Принципы

- Уточнение у пользователя возможно **только на шаге 1** (поиск матча)
- На остальных шагах: если подзапрос вернул мало данных — работаем с тем что есть, pipeline не останавливается
- Каждый шаг — отдельный модуль, который можно тестировать и править изолированно
- UI отображает прогресс: на каком шаге сейчас находится pipeline

## Ввод пользователя (до pipeline)

На UI пользователь **обязательно** выбирает:
1. **Вид спорта** — кнопки-табы (хоккей, футбол, баскетбол...)
2. **Лига** — выпадающий список зависит от выбранного спорта (КХЛ, НХЛ...)

Дата определяется автоматически (сегодня/ближайшая). Эти параметры передаются в шаг 1 и сужают поиск.

## Шаги оркестратора

```
Шаг 1: Поиск матча (Perplexity)
    ↓ (не нашли → уточнение у пользователя)
    ↓ (нашли → далее)
Шаг 2: Контекст и мотивация (Perplexity)
    ↓
Шаг 3: Форма команд — последние 5 матчей (Perplexity)
    ↓
Шаг 4: H2H — встречи в сезоне (Perplexity)
    ↓
Шаг 5: Статистика сезона (Perplexity)
    ↓
Шаг 6: Кадры и контекст (Perplexity)
    ↓
Шаг 7: Букмекерские линии (Perplexity)
    ↓
Шаг 8: Анализ (Claude Sonnet)
    ↓
Отображение отчёта
```

Шаги 2–7 могут выполняться параллельно (все — запросы в Perplexity, не зависят друг от друга).

## Работа с данными Perplexity

### Очистка ответов
Perplexity добавляет ссылки-цитаты в текст (`[1]`, `[3][5]`). **Каждый шаг обязан** очищать их при парсинге через `clean()` — удалять `[N]` паттерны из всех строковых полей.

### Подход: данные → UI → анализ
- **Фаза 3** (шаги 2–7): собираем структурированные данные из Perplexity, форматируем в `analysis.ts` в читаемые строки, передаём в UI. Аналитика Claude — заглушка.
- **Фаза 4** (шаг 8): Claude получает все данные, форматирует и анализирует за один вызов. Заглушки заменяются реальной аналитикой.

Каждый реализованный шаг сразу отображается на фронте с реальными данными — так можно проверить качество до подключения Claude.

## API

Точка входа: `POST /api/analyze`

```ts
// Вход
{ query: string, sport: string, league: string }

// Выход (стриминг прогресса + финальный результат)
{ step: number, status: "in_progress" | "done" | "error", data?: ... }
```

## Масштабирование

- Новая лига = новый конфиг. Не новый код
- Промпты Perplexity шаблонизированы — лига, сезон, метрики подставляются из конфига
- Системный промпт Claude — шаблон, роль аналитика и набор рынков из конфига

## После MVP: кэширование отчётов

Сейчас не реализуем, но проектируем с учётом:
- Каждый отчёт привязан к матчу (league + team1 + team2 + date = ключ)
- Если другой пользователь запросит тот же матч — отдаём готовый отчёт
- TTL: если до матча >24ч, отчёт может устареть — перегенерация контекстных блоков
