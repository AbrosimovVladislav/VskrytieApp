# Backend — Pipeline

## Архитектура

5 шагов: поиск матча → 3 шага сбора данных (параллельно) → анализ Claude.

```
Пользователь: команда + спорт + лига
        ↓
Шаг 1: Поиск матча (API-Sports)
        ↓ (не нашли → Perplexity для уточнения названия → retry API)
        ↓ (нашли → параллельно шаги 2–4)
        ↓
┌──────────────┬──────────────┬──────────────┐
│ Шаг 2        │ Шаг 3        │ Шаг 4        │
│ Коэффициенты │ Форма        │ Контекст     │
│ API-Sports   │ API-Sports   │ Perplexity   │
└──────┬───────┴──────┬───────┴──────┬───────┘
       └──────────────┴──────────────┘
                      ↓
Шаг 5: Claude Sonnet — анализ → AnalysisReport
                      ↓
            UI: отображение отчёта
```

## API-Sports Hockey v1

**Base URL**: `https://v1.hockey.api-sports.io`
**Auth**: header `x-apisports-key`
**Метод**: только GET

### Особенности Hockey API (отличия от Football API)

- **`games`** вместо `fixtures` — матчи называются "games"
- **`game`** вместо `fixture` — в параметрах odds
- **Нет `next`/`last`** — нельзя запросить "следующий матч" или "последние 5". Нужно получить все матчи сезона и фильтровать
- **Сезон = 4 цифры** — `2025`, не `2025/2026`
- **Статусы завершённых матчей**: `FT` (основное время), `AOT` (овертайм), `AP` (буллиты)

## Принципы

- **API-Sports для структурированных данных** — коэфы, результаты, счета. Точные, без галлюцинаций.
- **Perplexity только для контекста** — травмы, вратарь, ротация. То, что API не покрывает.
- Уточнение у пользователя — **только на шаге 1** (поиск матча)
- На остальных шагах: если данные неполные — работаем с тем что есть
- Шаги 2–4 выполняются параллельно
- Каждый шаг — отдельный модуль

## Шаг 1 — Поиск матча

**Источник**: API-Sports (`/teams?search={query}` → `/games?team={id}&league={id}&season={year}`)

Пользователь вводит название команды. Ищем team_id в API-Sports, затем из матчей сезона берём ближайший со статусом `NS`.

**Fallback**: если API не нашёл (неточное название) — запрос в Perplexity для уточнения → повторный поиск в API.

**Выход**: `MatchInfo` (team1, team2, game_id, date, time, league)

Подробнее: [`backend/step-find-match.md`](backend/step-find-match.md)

## Шаг 2 — Букмекерские линии

**Источник**: API-Sports (`/odds?game={game_id}`)

**Что собираем**: коэффициенты от букмекеров. Рынки: исход в основное время (П1/X/П2) + тотал (Б/М). Маппинг букмекеров API → названия из конфига лиги.

**Выход**: `OddsData`

Подробнее: [`backend/step-odds.md`](backend/step-odds.md)

## Шаг 3 — Форма команд

**Источник**: API-Sports (`/games?team={id}&league={id}&season={year}` → фильтр завершённых → последние 5)

**Что собираем**: последние 5 матчей каждой команды (дата, соперник, счёт, дома/выезд, результат).

**Выход**: `FormData`

Подробнее: [`backend/step-form.md`](backend/step-form.md)

## Шаг 4 — Контекст (кадры)

**Источник**: Perplexity API (единственный шаг с Perplexity)

**Что собираем**: травмы, дисквалификации, вратарь (основной/запасной), ожидаемая ротация, медиа-контекст за 2 недели.

**Выход**: `SquadContextData`

Подробнее: [`backend/step-squad-context.md`](backend/step-squad-context.md)

## Шаг 5 — Анализ Claude

Claude получает все данные (шаги 2–4) и формирует рекомендацию.

Логика рекомендации (иерархия):
1. **Коэффициенты = основа** — implied probability определяет базового фаворита
2. **Контекст = корректировка** — травмы, вратарь, ротация могут сдвинуть оценку, если букмекеры учли их с задержкой
3. **Форма = объяснение** — показывает *почему* линии такие

Claude выдаёт:
- Анализ каждой секции (data + analysis)
- Summary (общий вывод)
- 2 ставки: исход + тотал, каждая с confidence (high/medium/low) и reasoning

**Выход**: `AnalysisReport`

Подробнее: [`backend/step-analysis.md`](backend/step-analysis.md)

## API

Точка входа: `POST /api/analyze`

```ts
// Вход
{ query: string, sport: string, league: string }

// Выход (стриминг прогресса + финальный результат)
{ step: string, status: "in_progress" | "done" | "error", data?: ... }
```

## Масштабирование

- Новая лига = новый конфиг (букмекеры, рынки). Не новый код.
- API-Sports покрывает хоккей, футбол, баскетбол одним аккаунтом.
- Промпт Perplexity шаблонизирован — лига и дата подставляются.
- Промпт Claude — шаблон, роль аналитика из конфига.
